# AI Gateway Agent Instructions

本文件适用于整个 `ai-gateway` 仓库及其所有子目录。

## 文档同步是完成标准

每次修改代码、配置、依赖、数据库结构、API、测试或目录结构时，都必须同时检查：

- `ARCHITECTURE.md`
- `CODEBASE.md`

如果变更影响文档描述，必须在同一次任务中同步修改对应文档。实现已经改变但文档仍描述旧状态时，任务不得视为完成。

纯格式整理、拼写修正等不改变结构或行为的修改，可以不改文档正文，但仍需确认两份文档没有因此失真。不要把两份文档写成逐次变更日志，它们应始终描述当前代码状态。

## ARCHITECTURE.md 更新规则

出现以下任一变化时，更新 `ARCHITECTURE.md`：

- 新增、删除或调整模块、分层边界和模块职责。
- 修改模块依赖方向、基础设施依赖或关键第三方依赖。
- 修改 HTTP、SSE、WebSocket、鉴权、限流、Provider 调度或计费调用链。
- 修改幂等、事务、并发扣减、错误处理或数据一致性约束。
- 修改核心数据关系、配置边界、部署方式或运行时状态。
- 新增能力、移除能力，或改变“当前能力边界”中的限制。

架构图、时序图、状态图和文字说明必须与实际实现一致。

## CODEBASE.md 更新规则

出现以下任一变化时，更新 `CODEBASE.md`：

- 新增、删除、重命名或移动目录、文件、类、接口、枚举或 record。
- 新增、删除或改变承担业务职责的关键方法。
- 修改 Controller 路由、Service 职责、Provider Adapter、过滤器或协议适配器。
- 修改 Entity、Mapper、Mapper XML、SQL 迁移、配置项或测试覆盖范围。
- 修改常见维护入口，使原有文件或方法索引不再准确。

`CODEBASE.md` 应覆盖全部主代码类型、Mapper XML 和测试类型；构造器、getter、setter 等简单访问器无需逐项记录。

## 提交前检查

完成任务前执行以下检查：

1. 确认新增或重命名的类型、Mapper 和测试能在 `CODEBASE.md` 中找到。
2. 确认关键调用链、事务顺序、协议事件和能力边界已反映在 `ARCHITECTURE.md`。
3. 确认文档使用当前文件名、类名、方法名、接口路径和数据库字段。
4. 运行与改动范围匹配的测试，并执行 `git diff --check`。
5. 在最终说明中列出同步更新过的文档；若无需修改正文，说明已检查且无语义影响。
